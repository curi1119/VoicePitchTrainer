# UI 改修計画(2026-06 合意)

> 進捗: **P1 実装済み(2026-06-13)**。P2・P3 は未着手。

## 背景・課題

スマホでの操作性が悪い(開発者の実使用評価)。実測(390×844 = iPhone 相当、単音発声タブ):

| セクション | 高さ | 問題 |
|---|---|---|
| ヘッダ | 76px | 2行に折り返し |
| Pitch Meter カード | **491px(全体の42%)** | 最大の場所食い。針メーターは瞬間値しか見えず実用性が低い |
| タブ | 39px | — |
| モード操作 | 255px | ヒント文が過半 |
| 88鍵ピアノ | 197px | 25px幅の鍵はスマホで実用性低 |
| 合計 | **1177px ≒ 1.4画面** | 操作にスクロールが必要 |

## コンセプト

針メーターを**ピッチグラフ(時間 × 音高)**に置き換えて主役にし、他を圧縮して**スマホ縦持ち1画面(100dvh)に全操作を収める**。
**判定ロジック・config.ts の調整値・音声処理は変更しない**(見せ方のみの改修)。

## 決定事項(2026-06-13)

1. **グラフ縦軸は半音等間隔**(MIDI リニア)。音名ラベル+主要ラインに Hz 併記。どの音域でも「半音のずれ」が同じ見た目の幅になる
2. **アナログメーターと波形表示は撤去**。マイク動作確認用に小さなレベルインジケータのみ残す
3. **88鍵ピアノはスマホでは折りたたみ**(タップで展開)、デスクトップ(md 以上)は常時表示。グラフ左軸を縦ミニ鍵盤にし、ハイライト(検出音/目標音)と参照音のタップ再生を代替

## PitchGraph 仕様

- Canvas + 60Hz 描画。既存方針どおり **ref ハンドル経由で直接更新**(React 再レンダリングを通さない。docs/architecture.md「React との接続」)
- 横軸: 直近 N 秒(初期 6 秒、config に `GRAPH.WINDOW_SEC` として追加)。右端が現在
- 縦軸: MIDI 半音等間隔。表示レンジは声とターゲットを含むよう自動追従(急に飛ばさず lerp でパン)。初期レンジは選択中の出題範囲/開始音付近
- 左軸 = 縦ミニ鍵盤: 白鍵/黒鍵の帯 + C 音名ラベル。検出音(青)・目標音(オレンジ)をハイライト。タップで参照音再生(88鍵の代替)
- ターゲット帯: 単音発声 = target ± `OK_CENTS`、音階練習 = 現在音 ± `TOLERANCE_CENTS` を緑帯+中心線で表示
- 軌跡: 平滑化後 midiFloat の折れ線。帯内は緑/帯外はアンバー。無音は線を途切れさせる
- 数値表示(音名・Hz・セント)はグラフ隅にコンパクト表示(従来どおり 10Hz 間引き)
- 判定メッセージとホールドバーはグラフ直下に残す(文言・挙動は不変)
- マイクレベル: RMS の小さなインジケータをヘッダかグラフ隅に表示

## ワンスクリーン化仕様

- ルートを `100dvh` のグリッドに: `header / tabs / graph(1fr) / msg+hold / controls / (piano)`
- ヒント文(使い方の説明)は ⓘ ボタンのポップオーバー/折りたたみへ移動
- モード操作は1行に圧縮(音階練習はチップ列を含め最大2行)
- 88鍵: スマホは `<details>` 折りたたみ、md 以上は常時表示
- `viewport-fit=cover` + `env(safe-area-inset-*)` 対応
- タッチターゲットは 44px 目安
