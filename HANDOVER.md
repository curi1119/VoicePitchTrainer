## 1. プロジェクト概要

発声の音程を鍛えるトレーニングアプリ。アプリが再生した音をユーザーがマイクに向かって発声し、音程が正しいか判定する。

### 開発者(発注者)の前提
- 技術職だが、画像制作・音源制作はできない → **UI・音は全てコード生成**(画像/音声アセット不使用)が方針
- 音楽の専門知識はほぼない(A=440Hz を知っている程度)→ ドキュメントやUIでは音楽用語に簡単な補足を付けること
- 作業言語は日本語

## 3. プロトタイプの機能仕様(v0.10 / 同梱HTMLが実装)

単一HTMLファイル・依存ゼロ。**ここに書かれた仕様と調整値が現時点の「正」**であり、React移植時はこれを忠実に再現すること。

### 3.1 共通UI
- **ピッチメーター**: アナログチューナー風の針(±50セント、±10セントに緑ゾーン)。音名・オクターブ・周波数・セント値表示
- **マイク入力波形**: リアルタイム波形表示(Canvas)
- **88鍵ピアノ** (A0〜C8, MIDI 21〜108): 横スクロール。検出音=青ハイライト、目標音=オレンジ枠。クリックで参照音再生。Cの鍵盤にのみオクターブラベル
- **音色切替**: ビープ(サイン波) / 簡易ピアノ(倍音加算合成: 6倍音、わずかな不協和性、指数減衰)
- 音名表記は当面 C〜B + #(英語式)のみ

### 3.2 モード1: チューナー(自由練習)
発声をリアルタイム表示するだけ。判定なし。

### 3.3 モード2: 単音発声トレーニング
1. 出題範囲を選択: **男性向け G2〜G4 (MIDI 43-67) / 女性向け G3〜G5 (MIDI 55-79) / カスタム**(プルダウンで上下指定、初期値C3〜C5)。プリセット選択中は範囲プルダウンをdisabled
2. 「出題」→ 範囲内からランダムなMIDIノートを **3秒** 再生(`QUIZ_TONE_DUR`)。**再生終了+0.2秒後から判定開始**(マイクがアプリ音を拾う誤判定の防止)。再生中はメッセージで「♪ 出題音を再生中…」と表示
3. **出題ボタンは判定確定まで disabled**(連打でのリセット問題対策)
4. 発声中のリアルタイムフィードバック: 「✔ その音をキープ…!」/「↓ 高すぎます」/「↑ 低すぎます」+ホールド進捗バー
5. 判定:
   - **正解**: ターゲット±50セント以内を **1.5秒** (`OK_HOLD_MS=1500`) キープ。成功音(880→1318.5Hzの2音)
   - **不正解**: 外れた音程のまま **1.2秒** (`NG_HOLD_MS=1200`) 安定で確定。「正解は C4(あなたの声は A3 で低いでした)」形式で表示+下降音
   - キープ中に範囲を外れると holdMs は2倍速で減少(即ゼロにはしない)。無音時も減少
6. **不正解後のリトライ**: 問題は閉じず、正しい音を1.5秒キープできるまで練習継続可能。「✔ 正しい音 C4 が出せました!(記録は不正解のまま)」。スコアは変わらない
7. 「もう一度聞く」(再生+判定一時停止)、「答えを見てパス」(脱出口)、スコア表示(正解/出題)

### 3.4 モード3: 音階練習
1. パターン: `ドレミファソファミレド` [0,2,4,5,7,5,4,2,0] / `1オクターブ往復` [0,2,4,5,7,9,11,12,11,9,7,5,4,2,0]
2. 開始音(MIDI 36〜72、初期C3)、テンポ 40〜160 BPM(初期80)
3. **ラウンド開始前に基音のメジャートライアド**(根音+長3度+完全5度の和音)を **2拍** 再生し、**半拍** 空けて音階開始。調を掴むため(ボイトレのピアノ伴奏方式)
4. ガイド音は **1拍の95%(最低0.5秒)** 鳴らす ※短いと合成ピアノが「弦をはじいた」ような音になる不具合があった
5. 各音1拍。判定: 発声フレームのうちターゲット±60セント以内が `max(6, voiced*0.35)` フレーム以上で合格。音ごとに○×をチップ表示
6. 1周終了→結果表示→**半音上げて次ラウンド**。最高音がMIDI 96超 or 24ラウンドで自動停止
7. ガイド音OFF可(OFF時はトライアドもスキップ)。**ガイドON時はヘッドホン推奨**(マイクがアプリ音を拾うため)とUIに明記済み

## 4. 音声処理の技術詳細(重要・移植時に値を変えないこと)

### 4.1 マイク入力
- `getUserMedia({audio: {echoCancellation:false, noiseSuppression:false, autoGainControl:false}})`
- 入力 → **ローパスフィルタ(BiquadFilter, lowpass, 1200Hz)** → AnalyserNode
- `fftSize = 4096`(約85ms窓 @48kHz。低音=男性の声の分解能確保のため2048から拡大した経緯あり)

### 4.2 ピッチ検出: YIN法(自前実装)
- de Cheveigné & Kawahara (2002)。CMNDF(累積平均正規化差分関数)+ 閾値 `YIN_THRESHOLD=0.15` を最初に下回るラグを採用 + 放物線補間
- 検出レンジ: `F_MIN=55Hz`(A1) 〜 `F_MAX=1200Hz`
- RMSゲート: `rms < 0.015` で無音棄却
- 閾値未達時の救済: CMNDF全域最小が0.30未満なら採用
- **経緯**: 当初は自己相関法(ACF2+)だったが、男性の低い声(C3以下)で**第5倍音にロックしてE5等と誤検出**する障害が発生。YINに差し替え+LPF+窓拡大で解決した
- **検証実績**: 「基音が弱く(振幅0.05)3〜6次倍音が強い」人声模擬信号で C2〜C5 全て誤差0.1セント以内。処理時間 約2.3ms/フレーム。**この合成信号テストをVitestの回帰テストとして必ず移植すること**(テストコードの雛形はセッション内で作成済み、ロジックは下記)
  - 模擬信号: 倍音振幅 [0.05, 0.15, 0.30, 0.28, 0.25, 0.18]、位相 h*0.7、軽いノイズ±0.005

### 4.3 平滑化(感度調整。Firefox実機で「ちょうどよい」と確認済みの値)
人声のジッター/ビブラート(常時±20〜40セント)対策の多段構成:
1. **メディアンフィルタ**: 直近9フレーム(約150ms)
2. **EMA**: 係数0.18(時定数 ≒ 90ms)
3. **ジャンプ棄却**: メディアンとEMAの差が0.7半音超の場合、直近3フレームが安定(±0.7半音以内)していれば即追従(意図的な音程移動)、不安定なら棄却(オクターブ誤検出等のノイズ)
4. **音名ヒステリシス**: 表示中の音から±65セント離れて初めて音名切替(境界チラつき防止)
5. **発声開始ゲート**: 連続4フレーム(約65ms)検出で発声扱い(子音・瞬間ノイズ無視)
6. **終了ゲート**: 8フレーム(約130ms)までの途切れは直前値を保持
7. **針の追従**: lerp係数0.10(アナログメーター風のゆったり追従)
8. **数値テキスト更新**: 10Hzに間引き(チラつき防止)。周波数表示は平滑化後のMIDI値から逆算

### 4.4 音の合成
- ビープ: サイン波、attack 10ms / release 80ms
- 簡易ピアノ: 倍音 [1,2,3,4,5,6]、ゲイン [1,.45,.22,.11,.06,.03]、不協和性 `f*p*(1+0.0004*p²)`、指数減衰 `dur*(1.6 - i*0.15)`
- 成功音/不正解音(下降音)あり
- AudioContextはユーザー操作起点で生成・resume(iOS Safari対策として既に正しい構造)

## 5. バランス調整の履歴(再調整の参考)

| 項目 | 経緯 |
|---|---|
| 正解キープ時間 | 0.5秒 →(短すぎ)→ 2.0秒 →(長すぎ)→ **1.5秒で確定** |
| 不正解確定時間 | 0.8秒 → 1.5秒 → **1.2秒**(キープ時間に比例調整) |
| 出題音の長さ | 1秒 → **3秒**(ユーザー要望) |
| メーター感度 | 初版は敏感すぎ → 4.3の多段平滑化で「ちょうどよい」評価 |
| ガイド音長 | 最大0.35秒 → **1拍の95%**(ピチカート化の解消) |

## 7. 既知の制限・未解決事項

- 音階練習でガイド音ONかつスピーカー使用時は自己音の拾い込みが起こりうる(ヘッドホン推奨で運用中。将来はエコーキャンセル的な対策余地あり)
